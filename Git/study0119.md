### **Git**
- 분산 버전 관리 시스템
- 리누스 토르발즈
- 버전 관리
  - 변화를 기록하고 추적하는 것
  - **다음 버전으로 갈 때 이전 버전으로부터의 ``변경 사항`` 만 기록 한다.**
  - Git에서 버전은 Commit을 의미함 (수정 -> add -> commit 의 사이클)
  ---
  - 구글Docs '안녕하세요' 파일 실습 ->
  - **버전 1**
    - 아무것도 없는 페이지에서 안녕하세요 한 줄을 작성한 행위(사실)
  - **버전 2**
    - 버전 1 이후에 엔터를 치고 반갑습니다를 입력한 행위
  - **버전 3**
    - 버전 2 이후에 엔터를 치고 안녕히계세요를 입력한 행위
  ---
  - 과거로 돌리기 위해 이전 버전 복구 = 과거 변경사항 기록 확인 & 선택
  - 위 버전들의 의미를 이해해야 **Git**을 제대로 활용할 수 있음
- 코드의 '변경 이력'을 기록하고 '협업'을 원활하게 하는 도구
- Git의 3가지 영역 (p.76 Git의 동작 참고)
  - Working Directory : 실제 작업 중인 파일들이 위치하는 영역
    - git init을 실행하는 순간 그 폴더와 하위 폴더 전부 관리 영역 아래 들어옴
    - **주의** git 저장소간에 상하 관계가 들어가선 안된다. 해당 구역은 온라인에 올릴 수 없는 음영구역이 된다. 오프라인 폴더에 남아있는 경우 제외하면 복구 방법이 없다. 반드시 서로 독립적인 레벨 상에서 존재해야 한다.
    - git init을 진행하면 물리적으로 해당 경로에 .git 폴더가 숨김 처리로 생성된다.(보고싶다면 옵션에서 숨김 파일 보기) 환경설정 파일이기 때문에 숨김 처리한다.
  - Staging Area : WD에서 변경된 파일 중 다음 버전에 포함시킬 파일들을 선택적으로 추가하거나 제외할 수 있는 중간 준비 영역
    - git status
    - git status 입력해도 빨간 글씨가 나오는 것은 아직 해당 작업이 Working Directory에서 넘어오지 않았기 때문이다.
    - git status 입력 후 초록 글씨가 나오면 정상적으로 Staging Area에 넘어온 것
    - git status 변경사항이 없다면 아무것도 출력되지 않음
    - git add . -> 현재 디렉토리 전체 처리
    - git에서 명령어를 입력 후 아무 소식이 없는 것은 보통 좋은 편
  - Repository : 버전 이력과 파일들이 영구적으로 저장되는 영역 모든 버전과 변경 이력이 기록됨
    - git log
    - git commit -m "__" -> __ : 버전의 이름
    - 나의 컴퓨터를 공용으로 쓰지 않는다면 보통 git config --global 형태로 사용자 지정
    - git config --list
    - commit을 쌓는다고 표현
  ---
  - commit 실습 ->
  - **first commit**
    - 처음 작성한 samsung.png SSAFY.png 새 폴더 README.md가 생성
  - **second commit**
    - README.md에 한 줄을 작성
    - first commit에 의존하고 있음(심화 내용)
  ---
  - **주의** 바탕화면에 git init을 하면 안된다.
- . : 현재 디렉토리 전체를 의미
- .. : 상위 디렉토리를 의미
